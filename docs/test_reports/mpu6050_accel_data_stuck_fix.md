# MPU6050 加速度数据固定不变问题修复报告

- 日期: 2026-04-27
- 模块: `rpi_driver/gateway_monitor/gm.c`
- 状态: 已修复

## 问题描述

驱动加载正常，I2C 通信正常（WHO_AM_I 读取正确返回 0x68），但通过 IIO 接口读取三轴加速度原始数据时，X/Y/Z 三个轴始终返回同一个固定值，不随传感器姿态变化而更新。

## 根因分析

驱动 probe 阶段仅向 `PWR_MGMT_1 (0x6B)` 写入 `0x00` 来唤醒传感器。当模块反复卸载/加载时，MPU6050 未经历断电，芯片内部状态残留，导致数据寄存器锁定在旧值，不再刷新采样数据。

## 修复方案

在 probe 中唤醒传感器之前，先执行一次软复位：

```c
// 软复位: 置位 PWR_MGMT_1 的 DEVICE_RESET 位
regmap_write(regmap, 0x6B, 0x80);
msleep(100);  // 等待芯片完成内部复位

// 唤醒: 清除 SLEEP 位
regmap_write(regmap, 0x6B, 0x00);
msleep(50);   // 等待传感器稳定
```

写入 `0x80` 会触发 MPU6050 的 `DEVICE_RESET`，芯片将所有寄存器恢复为默认值并重新初始化内部信号通路，确保数据寄存器从干净状态开始采样。

## 涉及改动

| 文件 | 改动 |
|------|------|
| `gm.c` | 新增 `#include <linux/delay.h>` |
| `gm.c` | probe 中在写 `0x00` 唤醒前增加写 `0x80` 软复位 + `msleep` 等待 |
